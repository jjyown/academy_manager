# 출석관리앱 작업 계획서

## 프로젝트 목표
- 학생/수업 기준으로 출석 상태를 정확히 기록하고 조회한다.
- 교사 권한으로만 수정 가능하도록 접근 제어를 적용한다.
- 변경 이력을 남겨 추적 가능한 운영 상태를 유지한다.

## 작업 운영 원칙
- 작업 1개가 끝날 때마다 `docs/plan.md`, `docs/context.md`, `docs/checklist.md`를 즉시 동기화한다.
- 문서 업데이트가 끝나기 전에는 해당 작업을 완료로 간주하지 않는다.
- 롤플레잉/전문가 의견 요청이 있으면 관련 전문가 의견을 답변에 포함하고, 의사결정 반영 시 문서 3종에도 기록한다.
- 사용자가 전문가 구성을 지정하지 않아도 요청 맥락에 맞는 전문가를 자동 선택해 의견을 포함한다.

- 목표: 수납 누락 방지 + 세금 신고 준비 시간 단축 + 원장/학부모 커뮤니케이션 일관성 확보
- 1차 범위: 학생관리 핵심 정보 정비 + 수납 ledger(청구/수납/환불/미수금) + 월별 리포트/CSV
- 1차 제외: 카드사·PG 실시간 정산 연동, 전자세금계산서 자동발행, 고급 회계 분개 자동화

### 전문가 합의 기반 1주차 실행안
1) 데이터 모델
   - 학생: 재원상태, 시작일/종료일, 보호자 연락처, 반/코스 매핑
   - 수납: 청구월, 납부일, 수납수단(현금/계좌/카드), 공급가/세액, 증빙유형, 환불 여부/사유
2) UX/화면
   - 학생 상세에 수납 탭 고정(월별 ledger 타임라인)
   - 상태 뱃지(완납/부분/미납/환불) + 월마감 체크
3) 세무 편의 산출물
   - 월별 수납 합계, 미수금 리스트, 수단별 합계, 환불 내역 CSV

### 결제 채널 선정 기준(결제선생 vs Bizzle)
1) 통일 원칙: SMS 간편결제 + 현장결제를 한 채널에서 처리 가능하면 단일 채널 우선
2) 필수 검증 항목: 환불/부분취소, 정산주기, 수수료, 거래ID 추적, CSV 품질
3) 의사결정 방식: 1~2주 파일럿 후 월마감 대사 시간/오류율 기준으로 최종 선택
