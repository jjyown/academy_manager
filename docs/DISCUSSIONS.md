## 사용법

### 토의 시작 — 의뢰인 한 줄
> "[주제] 토의해서 진행해주세요. 모든 발언 [도메인 파일]에 기록"

자연어로도:
> "출석 화면에 결석 사유 입력 추가하는 거 토의해주세요"

→ Claude가 자동으로:
1. **도메인 판별** — 어느 도메인 파일에 누적할지 1차 결정 (불명확 시 의뢰인 확인)
2. 관련 페르소나 식별 (예: product-manager / ui-ux-designer / education-expert / academy-frontend-developer)
3. **라운드 1**: 각자 독립 의견 (다른 페르소나 의견 안 보고)
4. **라운드 2**: 다른 의견 보고 보충·반박
5. **라운드 3**: chief-reviewer 종합
6. 모든 발언을 해당 도메인 파일 상단에 시간순 누적
7. 의뢰인에게 결정 요청

### 토의 도중 의뢰인 개입 (선택)
관전 중 의견 추가하고 싶으면:
> "잠깐, 의뢰인 의견 — [...]"

→ `**HH:MM 의뢰인 발언**` 형식으로 기록 후 토의 계속.

### 토의 종결
의뢰인 결정 후:
> "이 토의 종결. [결론] 으로 처리. 구현은 [페르소나]가"

### 토의 재개
보류·폐기됐던 거 다시 열기:
> "[주제] 토의 재개. 추가 의견 받자"
→ 같은 섹션 하단에 `### YYYY-MM-DD 재개` 추가

### Archive 분리
도메인 파일이 길어지면 (50개 토의 초과 또는 분기 종료):
> "DISCUSSIONS_homework.md 2026 Q2 종결 토의들 archive로 분리해줘"
→ `docs/discussions-archive/homework-2026-Q2.md`로 이동

### 도메인 격리 룰 (2026-05-18 신설)
- **다른 도메인 내용을 같은 도메인 파일에 누적 금지**. 회의록 페이지 내용 겹침 회피.
- 도메인 판별 불명확 시 의뢰인에게 1차 확인.
- 본 인덱스에 시간순 색인 한 줄 추가 + 도메인 파일에 상세 누적.
- 새 채팅(작업창) 시작 시 의뢰인이 도메인 명시 선언하면 그 작업창에서는 해당 도메인 외 작업·검토 거절 (메모리 `feedback_chat_session_domain_declaration.md`).
